## Build order (see specs/build-plan.md)
Foundation S00–S03 → Metrics S04–S11 (one session each) → Frontend S12–S14 →
Ship S15–S16. Clear context + set model between sections.

## Notes
- `internal/` is Go-idiomatic for non-importable packages — keeps the public
  API surface clean for a single-binary tool.
- Query files are split per-metric (one file per build section) so each Claude
  Code session touches a small, isolated file — supports the no-compaction rule.
- The reference drafts (`config.go`, `db.go`) from planning are guidance only;
  S01/S02 build clean from spec.
